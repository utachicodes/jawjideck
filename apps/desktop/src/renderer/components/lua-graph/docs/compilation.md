# Compilation

## How the Compiler Works

The Lua compiler transforms your visual graph into ArduPilot-compatible Lua code through a multi-step pipeline.

### Pipeline

1. **Topological Sort** - Orders nodes so every node is evaluated after its inputs
2. **Variable Assignment** - Each output port gets a unique Lua variable name
3. **Code Generation** - Each node emits its Lua code using the assigned variable names
4. **State Variables** - Timing and edge-detection nodes get persistent state variables declared at the top
5. **Wrapping** - Everything is wrapped in the ArduPilot `function update() ... return update, interval end` pattern

### Generated Code Structure

```lua
-- Auto-generated by ArduDeck Lua Graph Editor
-- Graph: "My Script"

-- State variables (persist across update cycles)
local prev_state_1 = false
local timer_2 = 0

function update()
  -- Sensor reads
  local voltage_3 = battery:voltage(0)

  -- Logic & math
  local is_low_4 = voltage_3 < 11.1

  -- Actions (guarded by triggers)
  if is_low_4 then
    gcs:send_text(4, "LOW BATTERY")
  end

  return update, 1000
end

return update, 1000
```

## Run Interval

The run interval determines how often ArduPilot calls your script's `update()` function. It's set per-graph and defaults to 200ms.

| Interval | Use Case |
| --- | --- |
| 50-100ms | Fast sensor polling, servo control |
| 200ms | General purpose (default) |
| 500-1000ms | Slow monitoring, battery checks |
| 5000ms+ | Periodic logging, status reports |

> Shorter intervals use more CPU. ArduPilot enforces a minimum of ~50ms between script runs.

## Error Detection

The compiler catches these errors before generating code:

- **Disconnected action nodes** - Action nodes with no trigger connected
- **Circular dependencies** - Feedback loops that can't be sorted
- **Type mismatches** - Incompatible port connections
- **Missing required properties** - Properties that need a value

## Export Flow

1. Click **Export Lua** in the toolbar
2. The compiler validates and generates code
3. If errors are found, a dialog shows the error list
4. If successful, a save dialog opens to write the `.lua` file
5. Copy the file to your FC's SD card: `/scripts/your_script.lua`
6. Reboot the flight controller to start the script
